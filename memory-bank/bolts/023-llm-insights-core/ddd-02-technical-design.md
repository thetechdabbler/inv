# Technical Design — LLM Insights Core: Hybrid Projections, Guardrails & Enriched Audit

## Architecture Pattern

**Hexagonal Architecture** — same as bolt 022 and the existing insights unit.

Three orthogonal concerns, all fitting the existing layer structure:

1 - **Guardrail + Disclaimer**: Domain-layer pure functions (`disclaimer.ts`, `post-processing.ts`). Use cases call them before returning results.

2 - **Hybrid Projections**: New application-layer use case (`generate-hybrid-projection.ts`) replaces the existing `generateProjections()` function. The route is updated; the function signature from the use case is new.

3 - **Enriched Audit**: Infrastructure-layer changes. The `AuditingLLMGateway` decorator captures more data; Prisma schema and repositories are extended. Use cases are unaware of the enrichment.

```typescript
export const INSIGHT_DISCLAIMER =
  "⚠️ This insight is generated by an AI model based on your portfolio data " +
  "as of the most recent recorded valuation. Rates and returns are assumed, " +
  "not guaranteed. Market conditions, tax rules, and personal circumstances " +
  "may differ significantly from modelled assumptions. This output is for " +
  "informational purposes only and does not constitute financial advice. " +
  "Consult a SEBI-registered investment advisor for personalised guidance.";

export function appendDisclaimer(narrative: string): string {
  if (!narrative.trim()) return INSIGHT_DISCLAIMER;
  return `${narrative}\n\n${INSIGHT_DISCLAIMER}`;
}
```

2 - **`src/domain/insights/post-processing.ts`**

```typescript
export interface PostProcessingTransform {
  name: string;
  apply: (text: string) => string;
}

export const SAFE_FALLBACK =
  "Your portfolio data has been analysed. " +
  "Please consult your financial advisor for personalised recommendations.";

// Regex substitution: replaces investment product recommendations
export const GUARDRAIL_PIPELINE: PostProcessingTransform[] = [
  {
    name: "investmentRecommendationGuardrail",
    apply: (text) =>
      text.replace(
        /\b(invest in|buy|purchase|sell|switch to|recommend[a-z]*\s+(fund|stock|share|etf))\b/gi,
        "consider reviewing your allocation",
      ),
  },
];

export function applyPostProcessing(text: string): string {
  let result = text;
  for (const transform of GUARDRAIL_PIPELINE) {
    try {
      result = transform.apply(result);
    } catch (e) {
      console.warn(`[post-processing] Transform "${transform.name}" failed:`, e);
    }
  }
  return result.trim().length < 20 ? SAFE_FALLBACK : result;
}
```

```typescript
async function findDebugView(queryId: string): Promise<DebugAuditView | null> {
  const record = await prisma.lLMQuery.findUnique({
    where: { id: queryId },
    include: { response: true },
  });
  if (!record || !record.response) return null;
  return {
    auditId: record.id,
    insightType: record.insightType,
    prompt: record.prompt,
    responseText: record.response.responseText,
    modelUsed: record.response.modelUsed,
    promptTokens: record.response.promptTokens ?? null,
    completionTokens: record.response.completionTokens ?? null,
    durationMs: record.response.durationMs ?? null,
    templateId: record.templateId ?? null,
    templateVersion: record.templateVersion ?? null,
    success: record.response.success,
    errorMessage: record.response.errorMessage ?? null,
    createdAt: record.createdAt.toISOString(),
  };
}
```

## HybridProjectionResult — use case design

`generateHybridProjection(snapshot, gateway)`:

```text
1. Get template = getTemplate("projection-quality-review")
2. Build ctx = buildRenderContext(snapshot, { ... })
3. If snapshot.deterministicProjections exists:
     - Include projection summary in ctx.projectionsSummary
     - Set ctx.hasProjections = "true"
   Else:
     - Set ctx.projectionsSummary = "No projections available..."
     - Set ctx.hasProjections = "false"
4. prompt = renderTemplate(template, ctx)
5. response = await gateway.complete(prompt, {
     insightType: "projection-quality-review",
     templateId: template.id,
     templateVersion: template.version,
   })
6. rawNarrative = response.text.trim()
7. processedNarrative = applyPostProcessing(rawNarrative)
8. narrativeWithDisclaimer = appendDisclaimer(processedNarrative)
9. return {
     deterministicData: snapshot.deterministicProjections
       ? convert to ProjectionSeries format
       : null,
     llmNarrative: narrativeWithDisclaimer,
     assumptions: [],   // future: parse from LLM response
     disclaimer: INSIGHT_DISCLAIMER,
     modelUsed: response.modelUsed,
   }
```

## Use Case Post-Processing Pattern

All 4 text-output use cases follow this pattern after migration:

```typescript
// After LLM response:
const rawText = response.text.trim();
const processed = applyPostProcessing(rawText);
const withDisclaimer = appendDisclaimer(processed);
return { <result field>: withDisclaimer, ..., modelUsed: response.modelUsed };
```

For structured-output use cases (`analyzeRisk`, `recommendRebalancing`) where the LLM returns JSON, post-processing applies to the `narrative` / `description` string fields only — NOT to the full JSON blob (which would break parsing). Each use case applies post-processing after JSON parsing, to the relevant string fields.

## Security Design

1 - **Debug endpoint auth**: Same iron-session middleware as all other `/api/v1/*` routes (ADR-002). Raw prompts contain only portfolio aggregates — no PII beyond what the user already owns.

2 - **Guardrail scope**: Post-processing runs on the text before it is returned to the UI. The raw (pre-processing) text is stored in `LLMResponse.responseText` for audit purposes — intentional for debuggability.

3 - **Template injection**: Disclaimer and post-processing are applied server-side only; the client cannot skip them.

---

## NFR Implementation

1 - **Performance**: `durationMs` measurement has negligible overhead (one `Date.now()` call). Post-processing regex is O(n) on response length (~500 tokens max).

2 - **Backwards compatibility**: All new DB columns are nullable. Existing audit records (null telemetry fields) remain valid. The `generateProjections()` use case is deprecated but not deleted — the route is the only consumer and is updated.

3 - **Reliability**: Disclaimer and post-processing cannot throw (error handling inside `applyPostProcessing`). Audit enrichment failure is logged but does not fail the insight response (existing AuditingLLMGateway pattern).

---

## Error Handling

1 - **Projections unavailable**: `deterministicData: null` returned; LLM narrative states projections not available — explicit instruction in `projection-quality-review` template.

2 - **LLM failure on hybrid projection**: Propagates `LLMUnavailableError`; route returns 503.

3 - **Debug auditId not found**: Returns 404 `{ code: "NOT_FOUND", message: "Audit record not found" }`.

4 - **Post-processing removes all text**: `applyPostProcessing` returns `SAFE_FALLBACK` — never returns empty string.
